# Welcome to AIFred Pro — Getting Started

## What is AIFred?

AIFred Pro is your personal AI infrastructure assistant. It combines task management (Pulse), autonomous job execution (Nexus sessions), persona-based permissions, and security guards into a unified system that runs on your hardware.

## Task Lifecycle

Tasks flow through a managed pipeline:

```
open → scored → investigated → executed/manual → closed
```

- **open**: Created by you or imported from a plan
- **scored**: `task-score` job adds `auto:` and `risk:` labels
- **investigated**: `task-investigator` evaluates candidates for automation
- **executed**: `task-executor` runs safe items autonomously; complex items are manual
- **closed**: Completed with a reason logged

## Nexus Jobs

The dispatcher runs every 5 minutes via cron. It reads `registry.yaml` and launches jobs based on schedule, budget, and pre-check conditions.

| Job | Schedule | Purpose |
|-----|----------|---------|
| health-check | Every 6h | Quick infrastructure health check |
| task-score | On-demand | Label new tasks with auto/risk scores |
| task-investigator | Every 4h | Evaluate candidates for automation |
| task-executor | Every 6h | Execute safe automated tasks |
| weekly-cost-report | Monday 9am | Summarize job costs |
| doc-sync-check | Every 24h | Check if docs need updating |
| priority-review | Monday 7am | Flag stale/stuck tasks |

## Key Commands

```bash
# Task management
pulse list                              # All tasks
pulse list --status open                # Open tasks
pulse create "title" --label "domain:infrastructure"  # New task
pulse close <id> --reason "Done: ..."   # Complete task
pulse ready                             # Next actionable task

# Nexus jobs
.claude/jobs/dispatcher.sh --list       # List all registered jobs
.claude/jobs/dispatcher.sh --status     # Job run history
.claude/jobs/event-watcher.sh --status  # Event watcher state
```

## Creating Projects vs Tasks

- **Tasks**: Individual work items. Create with `pulse create`.
- **Projects**: Phased plans with dependencies. Define in YAML, import via `POST /api/v1/projects/import`. Tasks get `source:orchestration` and `phase:<id>` labels.

Use projects when work has multiple dependent phases. Use tasks for standalone items.

## Security

AIFred includes layered security guards:

- **Document Guard**: Protects critical files (.env, CLAUDE.md, settings.json) from accidental modification
- **Credential Guard**: Detects secrets in tool output and blocks leakage
- **Persona Guard**: Restricts what each headless persona can do (read-only vs full access)
- **Safety Rules**: Hard constraints in `.claude/jobs/rules/safety.yaml` — never delete user content, never push to git, etc.

## Profiles

Composable profile layers determine which hooks, permissions, and agents are active. Check your current profile with `/profile`. Available layers are defined in `profiles/`.

## Settings Files

- `settings.json`: Generated by the profile loader — don't edit manually
- `settings.local.json`: Your personal overrides (Pulse permissions, custom allow rules)
- Both matter for security hooks

## What to Expect

Jobs run automatically every 5 minutes. Here's what happens without your intervention:

1. `task-score` labels new tasks with automation readiness
2. `task-investigator` evaluates candidates and promotes or blocks them
3. `task-executor` runs safe items autonomously
4. You handle everything else — complex tasks, design decisions, reviews

## Resuming After Interruption

If setup is interrupted, next session Claude will re-run the hard gate checks.
Once infrastructure is confirmed healthy, it will automatically run:
```bash
pulse list --label source:orchestration --status open
```
to find remaining setup tasks and resume from where you left off.

## Prerequisites

The `claude` CLI must be on your PATH for headless execution. The dispatcher and executor invoke `claude -p` with persona configurations.
