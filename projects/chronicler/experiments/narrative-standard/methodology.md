# Narrative Gold-Standard: Methodology Document

**Subject:** Historical Figure 19639 — Minaro Autumnalsculpt the Windy
**World:** Tar Thran (The Land of Dawning), World ID 1
**Date:** 2026-03-03
**Purpose:** Establish a gold-standard biographical narrative methodology for Chronicler AI development

---

## 1. Overview

This experiment produces three biographical narratives of the same Dwarf Fortress historical figure, each generated by a different Claude model tier (Haiku, Sonnet, Opus). The goal is to:

1. **Formalize** a database-to-narrative workflow that Chronicler can eventually automate
2. **Benchmark** narrative quality and factual accuracy across model tiers
3. **Create a gold-standard** fact registry against which AI-generated narratives can be validated
4. **Investigate model-effect** on storytelling variance, depth, and accuracy

## 2. Database Search Methodology

### 2.1 Phase 1: Core Entity Extraction

**Objective:** Retrieve the primary record and all direct attributes.

| Step | Query Target | Purpose |
|------|-------------|---------|
| 1.1 | `historical_figures WHERE id = {hf_id}` | Core identity, race, caste, birth/death, flags (werebeast, author, deity, vampire, etc.) |
| 1.2 | `historical_figure_skills WHERE hf_id = {hf_id}` | All skills and IP values |
| 1.3 | `hf_links WHERE hf_id = {hf_id} OR target_hf_id = {hf_id}` | Bidirectional relationships (deity, master, apprentice, spouse, etc.) |
| 1.4 | `entity_links WHERE hf_id = {hf_id}` | Entity memberships (current, former, enemy) |
| 1.5 | `entity_position_links WHERE hf_id = {hf_id}` | Position history with dates |
| 1.6 | `site_links WHERE hf_id = {hf_id}` | Site associations (seat of power, residence) |
| 1.7 | `written_contents WHERE author_hf_id = {hf_id}` | Authored works (poems, compositions, etc.) |

### 2.2 Phase 2: Event Timeline Construction

**Objective:** Build the complete chronological event backbone.

| Step | Query Target | Purpose |
|------|-------------|---------|
| 2.1 | `historical_events WHERE details::text LIKE '%{hf_id}%' ORDER BY year` | All events mentioning the HF (as subject, object, witness) |
| 2.2 | Parse JSONB `details` for each event | Extract structured fields: attacker, victim, site, coords, injury, cause, etc. |
| 2.3 | Classify events by type | Group into: life changes, combat, kills, entity transitions, state changes, written works, intrigue |

### 2.3 Phase 3: Relationship Graph Expansion

**Objective:** Look up all referenced entities to provide narrative context.

| Step | Query Target | Purpose |
|------|-------------|---------|
| 3.1 | `historical_figures WHERE id IN ({referenced_hf_ids})` | Names, races, birth/death for all linked HFs (deities, masters, apprentices, victims, combatants) |
| 3.2 | `entities WHERE id IN ({referenced_entity_ids})` | Names and types for all referenced entities (civilizations, governments, groups) |
| 3.3 | `sites WHERE id IN ({referenced_site_ids})` | Names and types for referenced sites |
| 3.4 | `regions WHERE id IN ({referenced_region_ids})` | Names and types for referenced regions |

### 2.4 Phase 4: Analytical Derivation

**Objective:** Compute derived facts that don't exist in any single record.

| Derivation | Method |
|-----------|--------|
| Age calculations | `current_year - birth_year` |
| Duration spans | `end_event_year - start_event_year` |
| Kill statistics | Aggregate slayer_race, victim demographics, frequency |
| Poetry analysis | Categorize by deity reference, circumstance, temporal patterns |
| Career arc segmentation | Identify phase boundaries (peaceful→cursed→rampage→political) |
| Entity relationship patterns | Count and categorize enemy accumulation rate |

### 2.5 Phase 5: Fact Registry Compilation

**Objective:** Assign a unique ID to every verifiable claim.

- Format: `F{NNN}: {claim}. — Source: {table.column}`
- Categories: Identity, Skills, Relationships, Memberships, Positions, Chronological Events, Analytical Derivations
- Each fact traces to a specific database table and column
- Derived facts explicitly marked with derivation method

## 3. Formalized Workflow (for Chronicler AI Implementation)

```
INPUT: hf_id (Historical Figure ID)

STEP 1: EXTRACT
  → Core record (historical_figures)
  → Skills (historical_figure_skills)
  → Links (hf_links, entity_links, entity_position_links, site_links)
  → Works (written_contents)

STEP 2: TIMELINE
  → All events mentioning hf_id (historical_events)
  → Sort chronologically
  → Parse JSONB details for each event
  → Classify by event type

STEP 3: EXPAND
  → Look up all referenced HFs (names, races, dates)
  → Look up all referenced entities (names, types)
  → Look up all referenced sites/regions

STEP 4: DERIVE
  → Calculate ages, durations, frequencies
  → Identify career phases and turning points
  → Compute kill/combat statistics
  → Analyze written work patterns

STEP 5: REGISTER
  → Assign fact IDs (F001–FNNN)
  → Map each fact to source table(s)
  → Flag derived vs. direct facts

STEP 6: NARRATE
  → Feed fact registry + raw data to LLM
  → Instruct: chronicle voice, focus on pathos/tragedy/triumph
  → Require: narrative body + chronological appendix

STEP 7: VALIDATE
  → Cross-reference every narrative claim against fact registry
  → Tag: [FACT] / [PROBABLE] / [EMBELLISHMENT] / [CREATIVE LICENSE] / [WILD GUESS] / [FABRICATION]
  → Compute coverage matrix: facts used / facts omitted / claims fabricated
```

## 4. Experiment Design

### 4.1 Variables

| Variable | Values |
|----------|--------|
| **Independent:** Model tier | Haiku 4.5, Sonnet 4.6, Opus 4.6 |
| **Controlled:** Prompt | Identical prompt with full fact registry |
| **Controlled:** Temperature | Default (model default) |
| **Controlled:** Instructions | Same narrative guidelines for all three |
| **Dependent:** Narrative quality | Prose style, structure, emotional impact (subjective) |
| **Dependent:** Factual accuracy | % claims traceable to fact registry |
| **Dependent:** Coverage | % of fact registry represented in narrative |
| **Dependent:** Fabrication rate | % claims not in fact registry |
| **Dependent:** Creative quality | Embellishment sophistication, literary merit (subjective) |

### 4.2 Outputs Per Model

1. **Biographical Narrative** — Chronicle-voice prose (~2000–5000 words)
2. **Chronological Appendix** — Year-by-year factual timeline
3. **Annotated Narrative** — Same text with inline fact-tags
4. **Coverage Matrix** — Which facts were used, omitted, or fabricated

### 4.3 Post-Mortem Analysis (Per Model)

- Fact accuracy score: `facts_correct / total_claims`
- Coverage score: `facts_used / total_facts_available`
- Fabrication score: `fabricated_claims / total_claims`
- Embellishment index: `embellishments / total_claims`
- Creative quality assessment (qualitative)

## 5. Evaluation Criteria

### 5.1 Fact Tags (for Annotated Narrative)

| Tag | Definition | Example |
|-----|-----------|---------|
| `[FACT]` | Directly matches a fact in the registry | "She was born an elf" (F004) |
| `[PROBABLE]` | Reasonable inference from available data | "She likely lived among elves in her youth" (implied by F102-F103) |
| `[EMBELLISHMENT]` | Adds literary flavor to a true fact | "Her spear sang through the air" (F021 has spear skill, but singing is literary) |
| `[CREATIVE LICENSE]` | Plausible but unsupported narrative connection | "The curse awoke something dark within her" (curse is F172, darkness is interpretation) |
| `[WILD GUESS]` | Speculative with no direct support | "She regretted the loss of her poetic gift" (no evidence of regret) |
| `[FABRICATION]` | Contradicts the fact registry | "She killed 20 victims" (F255 says 12) |

### 5.2 Quality Dimensions

1. **Narrative Arc:** Does the story have a clear beginning, middle, and end with dramatic structure?
2. **Emotional Resonance:** Does the narrative evoke pathos, tragedy, or triumph?
3. **Factual Grounding:** How many claims are traceable to the fact registry?
4. **Creative Sophistication:** Are embellishments well-crafted and appropriate?
5. **Completeness:** Does the narrative cover the major life phases?
6. **Chronological Coherence:** Is the timeline internally consistent?

## 7. Implications for Chronicler Development

This experiment directly informs Phase 3 (Narrative Engine) of the Chronicler project:

1. **Workflow validation:** The 7-step formalized workflow (Extract→Timeline→Expand→Derive→Register→Narrate→Validate) serves as the blueprint for the automated narrative pipeline.
2. **Model selection:** Comparative results will inform which model tier to use for different narrative tasks (quick summaries vs. in-depth biographies).
3. **Prompt engineering:** The narrative prompt used here becomes the template for Chronicler's LLM interface.
4. **Quality baseline:** The coverage and accuracy scores establish the floor that automated narratives must meet or exceed.
5. **Fact tagging system:** The 6-level annotation taxonomy can be built into Chronicler's validation pipeline.
