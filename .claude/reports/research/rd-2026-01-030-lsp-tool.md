## Summary

The LSP tool is a native Claude Code feature that connects to Language Server Protocol servers, giving Claude type-aware code intelligence: jump to definitions, find references, hover type info, list symbols, find implementations, and trace call hierarchies. It also provides **automatic diagnostics** — after every file edit, the language server reports type errors, missing imports, and syntax issues back to Claude immediately.

This is a significant upgrade over grep-based code navigation, especially for typed languages.

## How It Works

1. Install a **code intelligence plugin** from the official marketplace
2. Ensure the **language server binary** is installed and in `$PATH`
3. Claude gains automatic diagnostics + code navigation

No configuration beyond plugin install. The LSP tool is not separately invokable — it's triggered automatically after edits (diagnostics) and available as a navigation tool.

## Benefits for Jarvis

### Automatic Diagnostics (Biggest Win)
After every file edit, Claude immediately sees type errors. Currently, Jarvis discovers type errors only when running tests or when the user reports them. With pyright-lsp:
- Missing imports caught on edit
- Type mismatches flagged immediately
- Undefined variable references detected
- Invalid function signatures caught

### Precise Code Navigation
- **Jump to definition**: No more grep-searching for function implementations
- **Find references**: Know everywhere a function is called before refactoring
- **Hover type info**: See function signatures without reading the file
- **Symbol listing**: Find all classes/functions in a file without reading it

### Multi-Language Coverage for Chronicler
The Chronicler project touches Python (main app), Lua (bridge scripts), JavaScript (hooks), and C# (LegendsViewer-Next reference). Four language plugins could cover the entire stack.

## Installation Plan

### Phase 1: Python (Immediate)
```bash
pip install pyright
/plugin install pyright-lsp@claude-plugins-official
```

### Phase 2: TypeScript + Lua (Short-term)
```bash
npm install -g typescript-language-server
brew install lua-language-server
/plugin install typescript-lsp@claude-plugins-official
/plugin install lua-lsp@claude-plugins-official
```

### Phase 3: C# (When needed)
```bash
dotnet tool install -g csharp-ls
/plugin install csharp-lsp@claude-plugins-official
```

## Caveats

- **Memory usage**: Language servers (especially pyright, rust-analyzer) can consume significant memory on large projects
- **Monorepo false positives**: May report unresolved imports for internal packages
- **Binary must be in $PATH**: Plugin won't work without the language server binary
- **Plugin persistence**: Plugins are per-scope (user/project/local) — install at user scope for cross-project availability

## Recommendation

**ADOPT** for Python and TypeScript immediately. These cover >95% of Jarvis's codebase interactions. Lua is a nice-to-have for DFHack bridge work. C# only needed when actively working on LegendsViewer-Next.

The automatic diagnostics feature alone justifies adoption — catching type errors on edit rather than at test time is a meaningful quality improvement.
